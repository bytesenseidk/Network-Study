1. Overview
==================================================
The internal network kill chain describes attacker behavior after gaining initial access.

It focuses on:

- expanding access
- maintaining persistence
- extracting data

This model helps:

- red teams simulate realistic attacks
- blue teams detect attacker progression

==================================================
3. Phase Breakdown
==================================================

Phase 1 — Initial Foothold

Entry point:

- phishing
- web exploit
- compromised credentials


Phase 2 — Host Enumeration

Gather information:

- OS version
- users
- running services


Phase 3 — Privilege Escalation

Gain higher privileges:

- local admin
- SYSTEM access


Phase 4 — Credential Access

Extract credentials:

- LSASS dump
- token extraction


Phase 5 — Lateral Movement

Move across network:

- SMB
- WinRM
- RDP


Phase 6 — Persistence

Maintain access:

- scheduled tasks
- backdoors
- Golden Tickets


Phase 7 — Defense Evasion

Avoid detection:

- log clearing
- obfuscation


Phase 8 — Command & Control

Establish communication:

- reverse shells
- beaconing


Phase 9 — Data Exfiltration

Extract sensitive data:

- file transfer
- cloud upload

8. Blue Team Detection
==================================================
Indicators include:

- unusual lateral movement
- abnormal credential usage
- persistence mechanisms
- outbound connections

Security teams should monitor:

- authentication logs
- network traffic
- endpoint activity
