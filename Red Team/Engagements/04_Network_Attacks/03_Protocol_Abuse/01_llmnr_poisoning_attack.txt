LLMNR Poisoning Attack
- A network attack that exploits the Link-Local Multicast Name Resolution (LLMNR) protocol to capture NTLM authentication credentials.

1. Overview
==================================================
LLMNR (Link-Local Multicast Name Resolution) is a protocol used in Windows networks to resolve hostnames when DNS resolution fails.

When a system cannot resolve a hostname using DNS, it broadcasts a request to the local network asking:

"Who has this hostname?"

Any system on the network can respond.

Attackers exploit this behavior by responding to these requests and pretending to be the requested resource.

If the victim attempts to authenticate to the attacker’s system, NTLM authentication credentials may be sent.

These credentials can then be captured and used for:

- offline password cracking
- pass-the-hash attacks
- lateral movement


==================================================
2. Attack Scenario
==================================================
Assume an attacker gains access to an internal corporate network.

A victim system attempts to access a shared resource using a hostname that cannot be resolved by DNS.

The victim system sends an LLMNR broadcast request to the local network.

The attacker responds claiming to be the requested host.

The victim attempts NTLM authentication to the attacker system.

The attacker captures the NTLM hash.


==================================================
3. Tools Required
==================================================

Responder
Common tool used to perform LLMNR poisoning and capture NTLM hashes.

Inveigh
PowerShell-based tool capable of performing LLMNR/NBT-NS spoofing.

Hashcat
Used to crack captured NTLM hashes offline.

Impacket
Used for pass-the-hash authentication using captured hashes.

==================================================
5. Step-by-Step Execution
==================================================
Step 1 — Gain Access to Internal Network

The attacker must be present on the same network segment as the victim.


Step 2 — Identify Network Interface

Example command:

ip a


Step 3 — Start Responder

Example command:

sudo responder -I eth0


Responder begins listening for LLMNR and NetBIOS name resolution requests.


Step 4 — Wait for Broadcast Requests

When a victim sends a name resolution broadcast, Responder replies pretending to be the requested service.


Step 5 — Capture NTLM Hash

Example output:

[SMB] NTLMv2-SSP Hash captured from DOMAIN\alice


Captured hashes are stored in the Responder logs directory.

7. Expected Results
==================================================
Successful LLMNR poisoning attacks may produce:

- NTLMv1 hashes
- NTLMv2 hashes
- domain user authentication attempts

These hashes can be used for:

- password cracking
- pass-the-hash attacks
- lateral movement


==================================================
8. Defensive Considerations
==================================================
Organizations can reduce the risk of LLMNR poisoning attacks through several controls.

Disable LLMNR

Disable LLMNR through group policy.

Disable NetBIOS Name Resolution

Prevent fallback name resolution protocols.

Network Segmentation

Limit broadcast domains.

SMB Signing

Require SMB signing to prevent relay attacks.

Security Monitoring

Detect abnormal name resolution responses.

10. Blue Team Detection
==================================================
LLMNR poisoning attacks generate abnormal name resolution traffic and suspicious authentication attempts.

Common indicators include:

- LLMNR broadcast queries receiving responses from non-authoritative hosts
- systems responding to name resolution requests that are not legitimate DNS servers
- NTLM authentication attempts directed to unusual hosts
- multiple authentication attempts following LLMNR queries
- broadcast name resolution activity across the network

Additional suspicious behavior may include:

- internal hosts impersonating file servers
- authentication attempts triggered by mistyped network resources
- systems performing unauthorized LLMNR or NetBIOS responses.

13. Mitigation Strategies
==================================================

Disable LLMNR

Prevent broadcast name resolution abuse.

Disable NetBIOS over TCP/IP

Eliminate fallback name resolution protocols.

Enforce SMB Signing

Reduce SMB relay attack risk.

Use Strong Authentication

Deploy multi-factor authentication where possible.

Segment Networks

Reduce broadcast domain exposure.

Monitor Authentication Activity

Detect abnormal NTLM authentication attempts.
