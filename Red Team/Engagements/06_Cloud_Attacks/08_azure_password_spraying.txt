Azure Password Spraying Attack
- A credential attack where attackers attempt common passwords across multiple Azure AD accounts to gain access without triggering account lockouts.

==================================================
1. Overview
==================================================
Password spraying is a technique where attackers test a small number of common passwords across many user accounts.

Unlike brute force attacks, password spraying:

- avoids account lockouts
- targets multiple users
- uses low-frequency attempts

Azure AD is a common target due to:

- cloud accessibility
- large user bases
- predictable usernames


==================================================
2. Attack Scenario
==================================================
An attacker identifies an organization domain:

example.com

The attacker enumerates valid usernames.

The attacker tests a common password such as:

Winter2024!

Across many accounts.

If any user uses a weak password, the attacker gains access.

==================================================
5. Step-by-Step Execution
==================================================
Step 1 — Enumerate Users

Example:

o365spray --enum -d example.com


Step 2 — Select Password

Example:

Winter2024!


Step 3 — Perform Password Spray

Example:

o365spray --spray -d example.com -p Winter2024!


Step 4 — Identify Valid Credentials

Successful logins are reported.


Step 5 — Access Azure Services

Attacker logs into:

- Microsoft 365
- Azure portal

10. Blue Team Detection
==================================================
Indicators include:

- multiple failed login attempts across many users
- login attempts from single IP targeting many accounts
- repeated use of same password pattern
- login attempts from unusual locations

Security teams should monitor:

- authentication logs
- failed login patterns
- geographic anomalies
