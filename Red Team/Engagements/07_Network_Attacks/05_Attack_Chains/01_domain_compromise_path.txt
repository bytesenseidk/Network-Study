==================================================
1. Overview
==================================================
A domain compromise path represents the sequence of steps an attacker takes to gain control over an Active Directory environment.

The goal is typically:

- Domain Administrator access
- control over authentication systems
- long-term persistence

Attackers rarely jump directly to domain admin.

Instead, they chain multiple techniques.

==================================================
3. Detailed Attack Phases
==================================================

Phase 1 — Initial Access

Common entry points:

- phishing
- exposed services
- web exploitation


Phase 2 — Local Privilege Escalation

Goal:

- gain SYSTEM or admin access on host


Phase 3 — Credential Access

Techniques:

- LSASS dumping
- SAM extraction
- browser credential theft


Phase 4 — Credential Reuse

Techniques:

- pass-the-hash
- pass-the-ticket


Phase 5 — Lateral Movement

Move to other systems:

- SMB
- WinRM
- RDP


Phase 6 — Active Directory Enumeration

Tools:

- BloodHound
- PowerView

Goal:

- identify privilege escalation paths


Phase 7 — Privilege Escalation (AD)

Examples:

- Kerberoasting
- AS-REP roasting
- ACL abuse


Phase 8 — Domain Controller Compromise

Access DC:

- extract KRBTGT
- perform DCSync


Phase 9 — Persistence

Techniques:

- Golden Ticket
- domain backdoor accounts

==================================================
5. Expected Results
==================================================
Successful domain compromise allows attackers to:

- control all domain systems
- access all user accounts
- disable security controls
- maintain persistence


==================================================
6. Defensive Considerations
==================================================

Privileged Access Management

Limit admin accounts.

Credential Protection

Use LSASS protection.

Network Segmentation

Limit lateral movement.

Monitoring

Detect abnormal activity.

Least Privilege

Reduce attack paths.

8. Blue Team Detection
==================================================
Indicators include:

- unusual authentication patterns
- lateral movement behavior
- privilege escalation events
- abnormal Kerberos activity

Security teams should correlate:

- logins across multiple systems
- credential usage anomalies
- privilege changes
