API Abuse Attack
- The exploitation of application programming interfaces (APIs) to perform unauthorized actions, extract data, or bypass security controls.

==================================================
1. Overview
==================================================
Modern applications rely heavily on APIs.

APIs expose functionality such as:

- user data access
- authentication
- transactions

API abuse occurs when attackers:

- misuse endpoints
- bypass client-side controls
- exploit weak validation
- manipulate requests

APIs often lack proper security controls compared to web interfaces.


==================================================
2. Attack Scenario
==================================================
A mobile app uses an API:

GET /api/v1/account/balance

The attacker interacts directly with the API and discovers:

- hidden endpoints
- missing rate limits
- weak authorization

The attacker extracts sensitive data or performs unauthorized actions.

==================================================
Step 1 — Discover API Endpoints

Sources:

- mobile apps
- JavaScript files
- traffic analysis


Step 2 — Intercept Requests

Use Burp Suite to capture API calls.


Step 3 — Modify Requests

Change:

- parameters
- headers
- tokens


Step 4 — Test Authorization

Attempt unauthorized access.


Step 5 — Automate Exploitation

Use scripts to extract data.

6. Common API Abuse Techniques
==================================================

Broken Authentication

Weak token validation.


IDOR via APIs

Manipulating object IDs.


Rate Limit Bypass

Brute force APIs.


Hidden Endpoints

Undocumented APIs.


Mass Assignment

Inject additional parameters.
