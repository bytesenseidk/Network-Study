Password Reset Abuse
- A technique where attackers exploit weaknesses in password reset mechanisms to gain unauthorized access to user accounts.

==================================================
1. Overview
==================================================
Password reset functionality is a critical part of authentication systems.

If improperly implemented, it can allow attackers to:

- reset passwords without authorization
- take over accounts
- bypass authentication controls

Password reset abuse is often easier than breaking authentication directly.


==================================================
2. Attack Scenario
==================================================
A web application provides a password reset feature.

The attacker identifies weaknesses such as:

- predictable reset tokens
- lack of validation
- weak verification steps

The attacker manipulates the reset process to gain access to another user’s account.


==================================================
3. Common Vulnerabilities
==================================================

Predictable Tokens

Reset tokens follow a predictable pattern.


Token Reuse

Tokens remain valid after use.


No Expiration

Tokens do not expire quickly.


Email Parameter Manipulation

Attacker changes the target email.


Missing Verification

No proper identity confirmation.

==================================================
6. Step-by-Step Execution
==================================================
Step 1 — Trigger Password Reset

Example:

POST /reset-password
email=victim@example.com


Step 2 — Intercept Request

Use Burp Suite to analyze parameters.


Step 3 — Modify Parameters

Example:

Change email:

victim@example.com → attacker@example.com


Step 4 — Analyze Reset Token

Check:

- length
- randomness
- expiration


Step 5 — Exploit Weakness

Examples:

- brute force token
- reuse token
- bypass validation


Step 6 — Reset Password

Set a new password for the victim account.

==================================================

==================================================
9. Defensive Considerations
==================================================

Secure Token Generation

Use cryptographically strong tokens.

Token Expiration

Expire tokens quickly.

Single Use Tokens

Invalidate after use.

Verification Controls

Require additional validation (MFA, email confirmation).

Rate Limiting

Prevent brute force attempts.
