Persistence
- Techniques used by attackers to maintain long-term access to compromised systems.

Stage in Attack Lifecycle:
Recon → Initial Access → Exploitation → Privilege Escalation → Lateral Movement → **Persistence** → Command & Control → Data Exfiltration

==================================================
1. Overview
==================================================
- Persistence refers to techniques attackers use to maintain access to systems even after reboots, credential changes, or partial remediation.
- After gaining control of a system, attackers often deploy mechanisms that allow them to reconnect later without repeating the entire attack chain.

Persistence allows attackers to:

- Maintain long-term access
- Survive system restarts
- Regain access if credentials are changed
- Continue operations without repeating the original compromise


==================================================
2. Purpose of Persistence
==================================================
Attackers establish persistence to ensure continued control of compromised environments.

Common goals include:

- Maintaining long-term access
- Avoiding detection
- Surviving system reboots
- Re-entering the system if removed
- Maintaining command channels


==================================================
3. Backdoors
==================================================
- A backdoor is a hidden method of accessing a system that bypasses normal authentication mechanisms.

Backdoors may include:

- Hidden user accounts
- Modified authentication systems
- Installed remote shells
- Malicious services

Backdoors allow attackers to regain access quickly after losing their original foothold.


==================================================
4. Startup Persistence
==================================================
- Startup persistence ensures malicious code executes automatically when a system starts.

Examples include:

- Startup scripts
- Boot configuration modifications
- System service modifications
- Autostart application entries

When the system boots, the malicious code runs automatically.


==================================================
5. Scheduled Tasks
==================================================
- Scheduled tasks allow attackers to execute malicious programs at regular intervals.

Common examples include:

- Cron jobs in Linux
- Windows scheduled tasks
- Automated maintenance scripts

Attackers may create tasks that:

- Reconnect to attacker infrastructure
- Reinstall malware
- Maintain remote access


==================================================
6. Service Persistence
==================================================
- System services often run with elevated privileges.

Attackers may create or modify services that:

- Launch malicious programs
- Start remote shells
- Execute scripts with elevated privileges

Because services start automatically, they provide reliable persistence.


==================================================
7. Web Shell Persistence
==================================================
- Web shells are malicious scripts installed on web servers that allow attackers to execute commands remotely.

Common characteristics include:

- Simple command execution interfaces
- Hidden or obfuscated code
- Disguised filenames

Web shells allow attackers to maintain access to compromised web servers.


==================================================
8. Credential Persistence
==================================================
- Attackers may maintain persistence by storing or reusing credentials.

Examples include:

- Creating hidden administrative accounts
- Reusing stolen credentials
- Adding new SSH keys
- Storing authentication tokens

These methods allow attackers to reauthenticate whenever needed.


==================================================
9. Linux Persistence Techniques
==================================================
Common persistence methods on Linux systems include:

- Cron job modification
- SSH authorized_keys abuse
- Startup script modification
- System service installation
- SUID binary manipulation

Linux persistence often relies on modifying system configuration files.


==================================================
10. Windows Persistence Techniques
==================================================
Common persistence techniques on Windows systems include:

- Registry run keys
- Scheduled tasks
- Windows services
- Startup folder entries
- DLL hijacking

Windows persistence often focuses on registry and service manipulation.


==================================================
11. Persistence Through Legitimate Tools
==================================================
- Attackers often use legitimate system tools to establish persistence.

This approach is often called:

Living-Off-the-Land

Examples include:

- PowerShell scripts
- System management tools
- Built-in administrative utilities

Because these tools are legitimate, their use may evade security detection.


==================================================
12. Persistence Workflow Example
==================================================

Initial Compromise
        ↓
Privilege Escalation
        ↓
Install Persistence Mechanism
        ↓
Verify Automatic Execution
        ↓
Maintain Long-Term Access


==================================================
13. Why Persistence Matters
==================================================
- Persistence allows attackers to maintain control over compromised systems.

Without persistence:

- Access may be lost after reboot
- Credentials may be reset
- Temporary exploits may disappear

Persistence ensures attackers can return whenever necessary.


==================================================
14. Questions & Answers
==================================================
1. Question:
What is persistence in offensive security?

Answer:
Persistence refers to techniques attackers use to maintain long-term access to compromised systems.

--------------------------------------------------
2. Question:
Why do attackers establish persistence?

Answer:
To maintain access even after system restarts or security changes.

--------------------------------------------------
3. Question:
What is a backdoor?

Answer:
A backdoor is a hidden access method that bypasses normal authentication mechanisms.

--------------------------------------------------
4. Question:
How do startup mechanisms provide persistence?

Answer:
They automatically execute malicious code when the system boots.

--------------------------------------------------
5. Question:
What role do scheduled tasks play in persistence?

Answer:
They allow attackers to run malicious programs automatically at set times.

--------------------------------------------------
6. Question:
What is a web shell?

Answer:
A web shell is a malicious script placed on a web server that allows remote command execution.

--------------------------------------------------
7. Question:
How can attackers maintain persistence using credentials?

Answer:
By creating hidden accounts or reusing stolen authentication credentials.

--------------------------------------------------
8. Question:
What is Living-Off-the-Land?

Answer:
Using legitimate system tools and features to maintain access while avoiding detection.

--------------------------------------------------
9. Question (Attack Perspective):
Why is persistence critical after privilege escalation?

Answer:
Because it allows attackers to retain access even if their initial exploit is patched.

--------------------------------------------------
10. Question (Defensive Perspective):
How can defenders detect persistence mechanisms?

Answer:
By monitoring new services, scheduled tasks, registry changes, and unusual system modifications.
