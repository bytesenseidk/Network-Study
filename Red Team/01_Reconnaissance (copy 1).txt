Reconnaissance
- The process of gathering information about targets, infrastructure, and potential attack paths before attempting exploitation.

Stage in Attack Lifecycle:
**Recon** → Initial Access → Exploitation → Privilege Escalation → Lateral Movement → Persistence → Command & Control → Data Exfiltration

==================================================
1. Overview
==================================================
- Reconnaissance is the first phase of an attack or penetration test.
- During this stage, attackers collect as much information as possible about a target organization, its systems, networks, and technologies.
- This information helps identify potential weaknesses and determine the most effective attack methods.


==================================================
2. Passive Reconnaissance
==================================================
- Passive reconnaissance gathers information about a target without directly interacting with the target systems.
- Because it does not generate traffic toward the target network, it is difficult to detect.

Passive techniques often rely on publicly available information sources.


==================================================
3. Open Source Intelligence (OSINT)
==================================================
- OSINT refers to information collected from publicly available sources.

Common OSINT sources include:

- Websites
- Social media platforms
- Search engines
- Public records
- Data leaks and breach databases
- Technical documentation

Attackers use OSINT to discover:

- Employee names
- Email address formats
- Technologies used by the organization
- Public infrastructure
- Business relationships


==================================================
4. DNS Enumeration
==================================================
- DNS enumeration involves collecting information from Domain Name System records.

DNS records can reveal important infrastructure details such as:

- Subdomains
- Mail servers
- Name servers
- IP addresses
- External services

This information helps attackers map an organization's external attack surface.


==================================================
5. Domain Intelligence
==================================================
- Domain intelligence focuses on analyzing domain ownership and infrastructure relationships.

Attackers may investigate:

- Domain registration details
- Associated domains
- Hosting providers
- Certificate records
- Historical DNS data

This can reveal additional domains and systems related to the target organization.


==================================================
6. Metadata Collection
==================================================
- Files published online may contain hidden metadata.

Common metadata sources include:

- PDF documents
- Microsoft Office files
- Images
- Public reports

Metadata may reveal:

- Author names
- Internal usernames
- Software versions
- File paths
- Internal system names

This information can provide insight into internal environments.


==================================================
7. Active Reconnaissance
==================================================
- Active reconnaissance involves directly interacting with target systems to gather technical information.
- Because it generates network traffic, it can potentially be detected by security monitoring systems.

Active reconnaissance is typically performed after passive reconnaissance has identified potential targets.


==================================================
8. Network Scanning
==================================================
- Network scanning identifies systems that are reachable on a network.

Attackers send probes to determine which hosts respond to network requests.

Network scanning helps attackers build a map of reachable systems.


==================================================
9. Host Discovery
==================================================
- Host discovery identifies which devices are currently active on a network.

Common host discovery techniques include:

- ICMP echo requests (ping)
- ARP scanning
- TCP probes
- UDP probes

These techniques help determine which systems are online and responding.


==================================================
10. Port Scanning
==================================================
- Port scanning identifies open ports on target systems.

Open ports often indicate services that may be running on a host.

Examples:

Port 80  → Web server (HTTP)  
Port 443 → Secure web server (HTTPS)  
Port 22  → SSH remote access  

Attackers scan ports to discover potential entry points.


==================================================
11. Service Enumeration
==================================================
- Service enumeration gathers detailed information about services running on open ports.

Attackers attempt to identify:

- Service type
- Configuration details
- Authentication mechanisms
- Potential vulnerabilities

Service enumeration provides a deeper understanding of the target system.


==================================================
12. Version Detection
==================================================
- Version detection attempts to identify the exact version of software running on a service.

Knowing the specific version allows attackers to:

- Search for known vulnerabilities
- Identify available exploits
- Determine patch levels


==================================================
13. Purpose of Reconnaissance
==================================================
- The goal of reconnaissance is to reduce uncertainty about the target environment.

By gathering information about systems, services, and technologies, attackers can:

- Identify weak points
- Map infrastructure
- Plan effective attack strategies
- Prioritize targets


==================================================
14. Example Reconnaissance Workflow
==================================================

1. Collect public information about the organization
2. Identify domains and subdomains
3. Discover externally accessible hosts
4. Scan for open ports
5. Identify running services
6. Determine software versions
7. Search for known vulnerabilities

15. Questions & Answers
==================================================
1. Question:
What is reconnaissance in offensive security?

Answer:
Reconnaissance is the process of gathering information about a target organization, its systems, and its infrastructure before attempting exploitation.

--------------------------------------------------
2. Question:
Why is reconnaissance important during an attack?

Answer:
It allows attackers to identify weaknesses and plan effective attack strategies.

--------------------------------------------------
3. Question:
What is passive reconnaissance?

Answer:
Passive reconnaissance collects information about a target without directly interacting with the target systems.

--------------------------------------------------
4. Question:
Why is passive reconnaissance difficult to detect?

Answer:
Because it does not generate network traffic toward the target systems.

--------------------------------------------------
5. Question:
What is OSINT?

Answer:
OSINT (Open Source Intelligence) is information gathered from publicly available sources such as websites, social media, and public records.

--------------------------------------------------
6. Question:
What type of information can attackers discover using OSINT?

Answer:
Employee names, technologies used by the organization, email formats, and infrastructure details.

--------------------------------------------------
7. Question:
What is DNS enumeration?

Answer:
DNS enumeration is the process of collecting information from DNS records to discover subdomains, servers, and infrastructure.

--------------------------------------------------
8. Question:
Why is DNS enumeration useful for attackers?

Answer:
Because it can reveal external services and potential entry points into a network.

--------------------------------------------------
9. Question:
What is metadata collection?

Answer:
Metadata collection involves extracting hidden information from files such as documents and images.

--------------------------------------------------
10. Question:
What information might metadata reveal?

Answer:
Author names, internal usernames, software versions, and system paths.

--------------------------------------------------
11. Question:
What is active reconnaissance?

Answer:
Active reconnaissance involves directly interacting with target systems to gather technical information.

--------------------------------------------------
12. Question:
Why is active reconnaissance easier to detect?

Answer:
Because it generates network traffic that may trigger monitoring or security alerts.

--------------------------------------------------
13. Question:
What is host discovery?

Answer:
Host discovery identifies which systems on a network are currently active.

--------------------------------------------------
14. Question:
What is port scanning?

Answer:
Port scanning identifies open ports on target systems that may expose running services.

--------------------------------------------------
15. Question:
Why are open ports important to attackers?

Answer:
Open ports indicate services that may contain vulnerabilities or misconfigurations.

--------------------------------------------------
16. Question:
What is service enumeration?

Answer:
Service enumeration gathers detailed information about services running on open ports.

--------------------------------------------------
17. Question:
What is version detection?

Answer:
Version detection identifies the exact version of software running on a service.

--------------------------------------------------
18. Question:
Why is version detection useful during reconnaissance?

Answer:
Because attackers can search for known vulnerabilities that affect that specific software version.

--------------------------------------------------
19. Question:
What is the primary goal of reconnaissance?

Answer:
To reduce uncertainty about the target environment.

--------------------------------------------------
20. Question (Attack Perspective):
Why do attackers perform reconnaissance before exploitation?

Answer:
Because understanding the target environment increases the chances of successful attacks.

--------------------------------------------------
21. Question (Defensive Perspective):
How can defenders detect reconnaissance activity?

Answer:
By monitoring unusual scanning activity, DNS queries, network probes, and abnormal traffic patterns.
