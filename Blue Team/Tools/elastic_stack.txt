Elastic Stack (ELK Stack)
- A powerful open-source platform used for centralized logging, data analysis, security monitoring, and SIEM capabilities.

==================================================
1. Overview
==================================================
Elastic Stack, commonly known as the **ELK Stack**, is a collection of tools used to collect, process, store, and visualize large volumes of data.

The stack consists of four primary components:

Elasticsearch  
Logstash  
Kibana  
Beats

Together, these tools allow security teams to build centralized logging platforms and perform advanced log analysis.

Elastic Stack is widely used in:

- Security Operations Centers (SOC)
- log management platforms
- SIEM deployments
- threat hunting environments
- incident response investigations

It allows analysts to detect suspicious activity by searching and analyzing logs from multiple sources across an environment.


==================================================
2. Elastic Stack Components
==================================================

Elasticsearch

A distributed search and analytics engine used to store and query large volumes of data.

Key capabilities:

- fast search queries
- scalable data storage
- distributed architecture
- indexing of structured and unstructured data


Logstash

A data processing pipeline used to collect, parse, and transform logs before storing them in Elasticsearch.

Functions include:

- ingesting logs from multiple sources
- parsing and normalizing log data
- filtering and enrichment


Kibana

A visualization and dashboard interface used to explore and analyze data stored in Elasticsearch.

Capabilities include:

- log searching
- dashboard visualization
- alert creation
- threat investigation


Beats

Lightweight data shippers installed on endpoints or servers to forward logs to Elasticsearch or Logstash.

Common Beats agents include:

Filebeat – log file collection  
Metricbeat – system metrics monitoring  
Packetbeat – network traffic monitoring  
Winlogbeat – Windows event log collection

        ↓

6. Example Detection Scenario
==================================================

Alert Trigger

Multiple failed login attempts are detected in authentication logs.


Investigation

Analysts search authentication logs in Kibana.


Analysis

Log data reveals a password spraying attack against several accounts.


Response

Security team blocks the attacking IP and resets affected credentials.

Elasticsearch, Logstash, Kibana, and Beats.

Which component stores and indexes data?
