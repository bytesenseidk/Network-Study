Addressing & Identification
- How devices, services, and communication endpoints are uniquely identified on a network


==================================================
1. Hardware Identification
==================================================
- Hardware identification refers to the unique identifiers assigned to network interfaces at the physical or data link layer.
- These identifiers allow devices on the same local network to recognize and communicate with each other.


MAC Addresses:
- A MAC (Media Access Control) address is a unique hardware identifier assigned to a network interface card (NIC).
- It is typically a 48-bit address written in hexadecimal format (for example: 00:1A:2B:3C:4D:5E).
- MAC addresses operate at the Data Link layer (Layer 2) and are used by switches to deliver frames within a local network.


==================================================
2. Network Identification
==================================================
- Network identification refers to logical addressing systems used to identify devices across multiple interconnected networks.
- These addresses enable routing and communication across the Internet and other large networks.


IP Addresses:
- An IP address is a logical identifier assigned to a device on a network that uses the Internet Protocol.
- IP addresses allow routers to determine where packets should be delivered.
- Unlike MAC addresses, IP addresses can change depending on the network the device connects to.


IPv4:
- IPv4 (Internet Protocol version 4) is the most widely used addressing system on the Internet.
- It uses 32-bit addresses written in dotted-decimal format (for example: 192.168.1.1).
- IPv4 supports approximately 4.3 billion unique addresses, which led to the development of IPv6 due to address exhaustion.


IPv6:
- IPv6 (Internet Protocol version 6) is the modern replacement for IPv4 and uses 128-bit addresses.
- Addresses are written in hexadecimal and separated by colons (for example: 2001:0db8:85a3::8a2e:0370:7334).
- IPv6 provides a vastly larger address space and improved support for modern networking features such as auto-configuration and improved routing.


==================================================
3. Service Identification
==================================================
- Service identification refers to how specific applications or services running on a device are distinguished from one another.


Ports:
- Ports are numerical identifiers used by transport protocols such as TCP and UDP to direct traffic to the correct service or application.
- Each network service listens on a specific port number so incoming traffic can be delivered to the correct process.


Well Known Ports:
- Well-known ports range from 0 to 1023 and are reserved for common network services.
- Examples include HTTP (port 80), HTTPS (port 443), DNS (port 53), and SSH (port 22).
- These ports are standardized so that clients know where to connect for common services.


Ephemeral Ports:
- Ephemeral ports are temporary ports assigned by the operating system when a client application initiates a network connection.
- They usually fall within the range of approximately 49152–65535.
- These ports allow multiple simultaneous connections between clients and servers.


==================================================
4. Communication Endpoints
==================================================
- Communication endpoints define the specific location where a network connection begins or ends.


Sockets:
- A socket represents the combination of an IP address and a port number.
- It uniquely identifies a communication endpoint used by an application.
- For example, a web server might listen on socket 192.168.1.10:80 to receive HTTP requests.
