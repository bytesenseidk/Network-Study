Routing & Packet Delivery
- How packets are forwarded between networks and across the global Internet


==================================================
1. Packet Forwarding
==================================================
- Packet forwarding refers to the process of moving network packets from a source device to a destination device across multiple interconnected networks.
- Routers analyze packet headers and use routing information to determine the best path for each packet.


Routing:
- Routing is the process of determining the path that packets take through a network from the source to the destination.
- Routers examine the destination IP address of each packet and decide where to forward it next.
- Routing enables communication between different networks and forms the foundation of the global Internet.


Routing Tables:
- A routing table is a data structure maintained by routers that contains information about available network paths.
- Each entry in the table typically includes a destination network, the next hop, and the interface used to forward packets.
- Routers consult their routing tables to determine the most efficient route for forwarding packets.


Default Gateway:
- A default gateway is the router that a device sends packets to when the destination network is outside its local network.
- If a device does not know the specific route to a destination, it forwards the packet to the default gateway.
- The gateway then determines the next step in delivering the packet toward its final destination.


==================================================
2. Routing Methods
==================================================
- Routing methods describe how routing information is configured and maintained within network devices.


Static Routing:
- Static routing uses manually configured routes that are defined by a network administrator.
- These routes remain fixed unless they are manually updated.
- Static routing is simple and predictable but does not adapt automatically to network changes or failures.


Dynamic Routing:
- Dynamic routing automatically learns and updates routing information using routing protocols.
- Routers exchange information about network paths and update their routing tables accordingly.
- This allows networks to adapt to changes such as link failures or new network segments.


==================================================
3. Routing Protocols
==================================================
- Routing protocols automatically learn and exchange routing information between routers to build and maintain routing tables.


RIP:
- Routing Information Protocol (RIP) is one of the earliest dynamic routing protocols.
- It uses hop count as its primary metric to determine the best route to a destination.
- RIP is simple but limited in scalability, making it suitable mainly for smaller networks.


OSPF:
- Open Shortest Path First (OSPF) is a link-state routing protocol commonly used in enterprise networks.
- It calculates the shortest path between routers using a topology map of the network.
- OSPF supports large and complex networks and converges faster than older protocols such as RIP.


BGP:
- Border Gateway Protocol (BGP) is the routing protocol that powers the global Internet.
- It is used to exchange routing information between large networks known as Autonomous Systems (AS).
- BGP determines how traffic is routed between Internet Service Providers and large organizations.


==================================================
4. Packet Movement
==================================================
- Packet movement describes the techniques used to transmit data across networks.


Packet Switching:
- Packet switching divides data into small packets that are transmitted independently across the network.
- Each packet may take a different route to reach the destination, where they are reassembled into the original message.
- Packet switching is efficient and resilient, making it the primary method used on the Internet.


Circuit Switching:
- Circuit switching establishes a dedicated communication path between two devices for the duration of a session.
- Traditional telephone networks used circuit switching to maintain a continuous connection.
- While reliable, circuit switching is less efficient than packet switching because the connection remains reserved even when no data is being transmitted.
