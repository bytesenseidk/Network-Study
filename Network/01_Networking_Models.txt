Networking Models
- Conceptual frameworks used to describe how network communication is structured and how protocols interact across layers.


==================================================
1. Purpose of Networking Models
==================================================
- Networking models provide a structured way to understand how different networking technologies work together.
- They divide complex communication processes into separate layers, each responsible for specific tasks.
- This layered approach simplifies design, troubleshooting, and protocol development.


==================================================
2. OSI Model Overview
==================================================
- The OSI (Open Systems Interconnection) model is a seven-layer conceptual framework used to describe how data moves through a network.
- Each layer performs specific functions and communicates with the layers above and below it.


==================================================
3. Layer 7 — Application Layer
==================================================
- The Application Layer provides network services directly to user applications.
- Protocols at this layer enable software such as web browsers, email clients, and file transfer programs to communicate over the network.

Examples:
- HTTP
- HTTPS
- FTP
- SMTP
- DNS


==================================================
4. Layer 6 — Presentation Layer
==================================================
- The Presentation Layer handles data formatting, encryption, and compression.
- It ensures that data sent by one system can be understood by another.

Examples:
- Encryption
- TLS / SSL
- Data encoding formats


==================================================
5. Layer 5 — Session Layer
==================================================
- The Session Layer manages communication sessions between devices.
- It establishes, maintains, and terminates connections between applications.

Examples:
- Session management
- Authentication sessions


==================================================
6. Layer 4 — Transport Layer
==================================================
- The Transport Layer ensures reliable communication between hosts.
- It controls data segmentation, error handling, and flow control.

Examples:
- TCP
- UDP


==================================================
7. Layer 3 — Network Layer
==================================================
- The Network Layer is responsible for logical addressing and routing packets across multiple networks.

Examples:
- IP
- IPv4
- IPv6
- ICMP


==================================================
8. Layer 2 — Data Link Layer
==================================================
- The Data Link Layer manages communication between devices on the same local network.
- It handles MAC addressing and frame delivery.

Examples:
- Ethernet
- ARP
- MAC addresses


==================================================
9. Layer 1 — Physical Layer
==================================================
- The Physical Layer defines how raw bits are transmitted across the physical medium.
- It includes cables, connectors, and electrical or optical signals.

Examples:
- Copper cabling
- Fiber optics
- Wireless signals


==================================================
10. TCP/IP Model Overview
==================================================
- The TCP/IP model is the practical networking model used by the Internet.
- It is simpler than the OSI model and groups several OSI layers together.


==================================================
11. TCP/IP Application Layer
==================================================
- The TCP/IP Application Layer combines the OSI Application, Presentation, and Session layers.
- It includes protocols used directly by applications.

Examples:
- HTTP
- HTTPS
- DNS
- SMTP
- FTP


==================================================
12. TCP/IP Transport Layer
==================================================
- The Transport Layer provides end-to-end communication between hosts.

Examples:
- TCP
- UDP


==================================================
13. TCP/IP Internet Layer
==================================================
- The Internet Layer corresponds to the OSI Network Layer and is responsible for addressing and routing.

Examples:
- IP
- IPv4
- IPv6
- ICMP


==================================================
14. TCP/IP Network Access Layer
==================================================
- The Network Access Layer combines the OSI Data Link and Physical layers.
- It handles hardware addressing and physical data transmission.

Examples:
- Ethernet
- MAC addressing
- Network Interface Cards (NIC)


==================================================
15. OSI vs TCP/IP Comparison
==================================================

OSI Model (7 layers)

7 — Application  
6 — Presentation  
5 — Session  
4 — Transport  
3 — Network  
2 — Data Link  
1 — Physical  


TCP/IP Model (4 layers)

4 — Application  
3 — Transport  
2 — Internet  
1 — Network Access  


==================================================
16. Layer Mapping
==================================================

OSI Layer 4     → TCP/IP Transport  
OSI Layer 3     → TCP/IP Internet  
OSI Layer 2–1   → TCP/IP Network Access

19. Why These Models Matter
==================================================
- Networking models help engineers understand where protocols operate and how different technologies interact.
- They are extremely useful when troubleshooting network problems or analyzing packet captures.
- They also provide a common language for engineers, developers, and security professionals when discussing networking concepts.

How might attackers exploit weaknesses in different networking layers?
