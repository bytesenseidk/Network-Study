Private vs Public IP Addresses
- How devices are addressed within local networks and across the global Internet.


==================================================
1. Overview
==================================================
- IP addresses are used to identify devices on a network.
- Some IP addresses are used internally within private networks, while others are used on the public Internet.
- This distinction allows organizations to manage internal networks while still connecting to the global Internet.


==================================================
2. Public IP Addresses
==================================================
- A public IP address is globally unique and reachable across the Internet.
- These addresses are assigned by Internet Service Providers (ISPs) or regional internet registries.
- Public IP addresses allow devices and services to communicate with systems across the global Internet.

Examples:

8.8.8.8
1.1.1.1
142.250.190.78

Common uses:
- Web servers
- Email servers
- Cloud services
- Internet gateways


==================================================
3. Private IP Addresses
==================================================
- Private IP addresses are used within local networks and are not directly reachable from the Internet.
- They allow organizations and home networks to assign addresses internally without consuming public IP space.
- Private addresses can be reused across different networks because they are not globally routed.


==================================================
4. RFC1918 Private Address Ranges
==================================================
Private address ranges were defined by RFC1918.

These ranges are reserved for internal networks:

Class A Private Range
10.0.0.0 – 10.255.255.255
(10.0.0.0/8)

Class B Private Range
172.16.0.0 – 172.31.255.255
(172.16.0.0/12)

Class C Private Range
192.168.0.0 – 192.168.255.255
(192.168.0.0/16)


Common home network example:

Router IP:
192.168.1.1

Devices:
192.168.1.10
192.168.1.15
192.168.1.25


==================================================
5. Why Private IP Addresses Exist
==================================================
Private IP addressing helps solve several networking challenges.

IPv4 Address Conservation
- The IPv4 address space is limited, so private networks reduce the need for public addresses.

Network Organization
- Organizations can design internal addressing schemes.

Security
- Private addresses are not directly reachable from the Internet.

Scalability
- Millions of internal devices can use private addressing.


==================================================
6. Internet Routing Rules
==================================================
Routers on the public Internet do not route private IP addresses.

Example:

192.168.1.10 → not routable on Internet  
10.5.3.2 → not routable on Internet  

Public addresses such as:

8.8.8.8 → routable globally

This ensures internal networks remain isolated from the public Internet.

8. Private vs Public Address Comparison
==================================================

Private IP
- Used inside internal networks
- Not routable on the Internet
- Reusable across networks

Public IP
- Globally unique
- Routable across the Internet
- Assigned by ISPs

==================================================
Understanding private and public IP addressing is important for:

Network design
Firewall configuration
Cloud networking
Security analysis
