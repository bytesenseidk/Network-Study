Protocol Foundations
- Core protocols that define the rules for communication across the Internet


==================================================
1. Internet Protocols
==================================================
- Internet protocols define how data packets are addressed and routed across networks.
- They form the foundation of communication on the Internet and allow devices on different networks to exchange information.


IP:
- The Internet Protocol (IP) is the core protocol responsible for addressing and delivering packets between devices across networks.
- IP determines how packets are structured and routed from a source device to a destination device.
- It operates at the Network Layer (Layer 3) of the OSI model and is responsible for logical addressing and packet forwarding.


IPv4:
- IPv4 (Internet Protocol version 4) is the most widely deployed version of the Internet Protocol.
- It uses 32-bit addresses to identify devices and networks and is typically written in dotted-decimal notation (for example: 192.168.1.1).
- IPv4 includes features such as packet fragmentation and header-based routing.


IPv6:
- IPv6 (Internet Protocol version 6) was developed to replace IPv4 due to the exhaustion of IPv4 addresses.
- It uses 128-bit addresses written in hexadecimal format separated by colons (for example: 2001:db8::1).
- IPv6 provides a vastly larger address space, improved routing efficiency, and built-in support for features such as auto-configuration and IPsec.


==================================================
2. Transport Protocols
==================================================
- Transport protocols provide end-to-end communication between applications running on different hosts.
- They manage how data is delivered, including reliability, ordering, and error handling.


TCP:
- Transmission Control Protocol (TCP) is a connection-oriented protocol that provides reliable data transmission.
- Before sending data, TCP establishes a connection using a process called the three-way handshake.
- It ensures that packets arrive in order, retransmits lost packets, and uses flow control and congestion control to maintain network stability.


UDP:
- User Datagram Protocol (UDP) is a connectionless transport protocol designed for speed and efficiency.
- Unlike TCP, it does not establish a connection or guarantee packet delivery.
- UDP is commonly used in applications where low latency is more important than reliability, such as streaming, online gaming, and DNS queries.


==================================================
3. Control Protocols
==================================================
- Control protocols support the operation and management of networks by providing diagnostic, error reporting, and address resolution functions.


ICMP:
- Internet Control Message Protocol (ICMP) is used for network diagnostics and error reporting.
- Routers and hosts use ICMP messages to communicate issues such as unreachable hosts or expired packet lifetimes.
- Tools such as ping and traceroute rely on ICMP to test connectivity and measure network paths.


ARP:
- Address Resolution Protocol (ARP) is used to map IP addresses to MAC addresses within a local network.
- When a device wants to communicate with another device on the same network, it sends an ARP request to discover the MAC address associated with the destination IP address.
- This process allows devices to correctly deliver frames at the data link layer.


==================================================
4. Infrastructure Protocols
==================================================
- Infrastructure protocols provide essential services that enable networks to function efficiently and automatically.


DNS:
- Domain Name System (DNS) translates human-readable domain names into IP addresses.
- For example, when a user enters a website address such as example.com, DNS resolves it to the corresponding IP address used by network devices.
- DNS operates as a distributed hierarchical system and is essential for navigating the Internet.


DHCP:
- Dynamic Host Configuration Protocol (DHCP) automatically assigns IP configuration settings to devices when they join a network.
- These settings typically include the IP address, subnet mask, default gateway, and DNS servers.
- DHCP simplifies network management by eliminating the need to manually configure network settings on every device.
