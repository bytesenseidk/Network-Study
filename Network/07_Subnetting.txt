Subnetting
- The process of dividing a larger IP network into smaller logical networks called subnets.


==================================================
1. Overview
==================================================
- Subnetting is used to divide a network into smaller segments to improve organization, security, and routing efficiency.
- Each subnet functions as its own logical network with its own range of IP addresses.
- Subnetting is commonly used in enterprise networks to separate departments, services, or security zones.


==================================================
2. Why Subnetting Is Important
==================================================
Subnetting provides several benefits:

Network Organization
- Large networks can be divided into smaller, manageable segments.

Improved Security
- Segments can be isolated using firewalls or access controls.

Reduced Broadcast Traffic
- Broadcast traffic is limited to individual subnets instead of the entire network.

Efficient IP Address Usage
- IP address ranges can be allocated based on network size requirements.


==================================================
3. IPv4 Address Structure
==================================================
An IPv4 address consists of **32 bits** divided into four octets.

Example:

192.168.1.10

Binary representation:

11000000.10101000.00000001.00001010

Each octet contains **8 bits**.

Total bits:

32 bits = IPv4 address


==================================================
4. Network vs Host Portion
==================================================
Every IP address contains two parts:

Network Portion
- Identifies the network.

Host Portion
- Identifies the specific device on that network.

Example:

192.168.1.10 /24

Network portion:
192.168.1

Host portion:
10


==================================================
5. Subnet Mask
==================================================
A subnet mask determines which portion of an IP address represents the network and which portion represents hosts.

Example subnet mask:

255.255.255.0

Binary representation:

11111111.11111111.11111111.00000000

The **1s represent the network portion**  
The **0s represent the host portion**


==================================================
6. CIDR Notation
==================================================
CIDR (Classless Inter-Domain Routing) expresses the subnet mask as the number of network bits.

Example:

/24

Means:

24 bits for network  
8 bits for hosts


Example mapping:

/24 → 255.255.255.0  
/16 → 255.255.0.0  
/8  → 255.0.0.0


==================================================
7. Subnet Address Components
==================================================
Every subnet contains several important addresses.

Network Address
- Identifies the subnet itself.

First Usable Host
- First assignable address.

Last Usable Host
- Last assignable address.

Broadcast Address
- Used to send traffic to all devices in the subnet.


Example:

Network: 192.168.1.0/24

Network Address:
192.168.1.0

First Host:
192.168.1.1

Last Host:
192.168.1.254

Broadcast Address:
192.168.1.255


==================================================
8. Number of Hosts per Subnet
==================================================
The number of usable hosts is calculated using:

2^n - 2

Where:

n = number of host bits

Example:

/24 network

Host bits = 8

2^8 = 256 addresses

Usable hosts:

256 - 2 = 254


==================================================
9. Example Subnet Calculation
==================================================
Network:

192.168.1.0/24

Divide into **two subnets**

New prefix:

/25

Subnet mask:

255.255.255.128


Resulting subnets:

Subnet 1
192.168.1.0/25

Range:
192.168.1.1 – 192.168.1.126

Broadcast:
192.168.1.127


Subnet 2
192.168.1.128/25

Range:
192.168.1.129 – 192.168.1.254

Broadcast:
192.168.1.255

==================================================
13. Why Subnetting Matters for Security
==================================================
Subnetting plays an important role in network security.

Benefits include:

Network Isolation
- Limits communication between segments.

Reduced Attack Surface
- Attackers cannot easily reach every system.

Access Control
- Firewalls can control traffic between subnets.

Monitoring
- Security tools can analyze traffic between network segments.
