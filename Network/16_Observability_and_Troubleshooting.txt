Observability & Troubleshooting
- How network activity is monitored, analyzed, and diagnosed when problems occur


==================================================
1. Diagnostics
==================================================
- Diagnostic tools help verify connectivity, measure network paths, and identify problems between devices.
- They are often the first tools used when troubleshooting network issues.


PING:
- PING is a diagnostic tool that tests whether a host is reachable over a network.
- It works by sending ICMP Echo Request packets to a destination and waiting for Echo Reply responses.
- Ping measures round-trip time and can help detect connectivity problems, high latency, or packet loss.


traceroute:
- Traceroute is a tool used to map the path that packets take from a source device to a destination.
- It works by sending packets with gradually increasing Time-To-Live (TTL) values and observing which routers respond along the path.
- Traceroute helps identify routing paths and locate points of delay or failure in a network.


netstat:
- Netstat is a utility used to display active network connections, listening ports, routing tables, and interface statistics.
- It helps administrators understand which services are running on a system and how they are communicating over the network.
- Although netstat is widely used, modern systems often replace it with tools such as "ss".


==================================================
2. Traffic Analysis
==================================================
- Traffic analysis involves capturing and inspecting network packets to understand how data flows through the network.


Packet Capture:
- Packet capture is the process of recording network traffic for analysis.
- Captured packets can reveal details about protocols, connections, errors, and potential security threats.
- Packet captures are often used during troubleshooting, incident response, and network performance analysis.


Wireshark:
- Wireshark is a graphical packet analysis tool used to inspect captured network traffic in detail.
- It can decode hundreds of network protocols and display packet structures in a human-readable format.
- Wireshark is widely used for troubleshooting network issues, analyzing protocols, and investigating suspicious traffic.


tcpdump:
- Tcpdump is a command-line packet capture tool commonly used on Linux and Unix systems.
- It allows administrators to capture and filter network traffic directly from the terminal.
- Tcpdump is powerful for quick diagnostics, remote troubleshooting, and automated traffic analysis.


==================================================
3. Monitoring
==================================================
- Monitoring involves continuously collecting data about network activity to ensure systems operate correctly and to detect issues early.


Logs:
- Logs are records generated by systems, applications, and network devices that document events and activities.
- Network logs can include connection attempts, errors, authentication events, and traffic information.
- Logs are essential for troubleshooting, auditing, and security investigations.


Metrics:
- Metrics are numerical measurements that track the performance and health of network systems.
- Examples include bandwidth usage, latency, packet loss, and CPU utilization on networking devices.
- Metrics help administrators monitor trends and detect abnormal behavior.


Network Monitoring:
- Network monitoring is the continuous observation of network devices, traffic, and performance metrics.
- Monitoring systems collect data from routers, switches, servers, and applications to identify failures or performance issues.
- Modern monitoring platforms can generate alerts when abnormal conditions occur, allowing administrators to respond quickly.


==================================================
4. Example Troubleshooting Workflow
==================================================

Basic Network Troubleshooting Steps

1. Verify connectivity using ping.  
2. Identify the network path using traceroute.  
3. Check active connections and listening services using netstat or ss.  
4. Capture and inspect packets using tcpdump or Wireshark.  
5. Review system and network logs for errors.  
6. Analyze metrics and monitoring alerts for abnormal behavior.
