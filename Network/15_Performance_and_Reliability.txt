Performance & Reliability
- How network performance is measured and optimized to ensure reliable communication


==================================================
1. Network Characteristics
==================================================
- Network characteristics describe measurable properties of how data travels across a network.
- These metrics help administrators understand network performance, diagnose problems, and optimize communication between systems.


Bandwidth:
- Bandwidth refers to the maximum amount of data that can be transmitted over a network link within a given period of time.
- It is typically measured in bits per second (bps), such as Mbps or Gbps.
- Higher bandwidth allows more data to be transmitted simultaneously, improving the potential capacity of the network.


Latency:
- Latency is the time it takes for data to travel from the source to the destination across a network.
- It is usually measured in milliseconds (ms).
- High latency can cause noticeable delays in applications such as online gaming, voice communication, and real-time systems.


Throughput:
- Throughput represents the actual rate at which data is successfully transmitted across the network.
- While bandwidth describes the theoretical maximum capacity, throughput reflects the real-world performance after factors such as congestion, overhead, and packet loss are considered.


Jitter:
- Jitter refers to variations in packet arrival time during transmission.
- Inconsistent delays can disrupt applications that rely on steady data streams, such as video conferencing or Voice over IP (VoIP).
- Low jitter is important for maintaining smooth real-time communication.


Packet Loss:
- Packet loss occurs when data packets fail to reach their destination during transmission.
- This can happen due to network congestion, faulty hardware, signal interference, or routing issues.
- Packet loss can degrade performance and cause retransmission, which increases latency and reduces throughput.


==================================================
2. Quality Control
==================================================
- Quality control mechanisms help manage network traffic to ensure important data receives priority and network resources are used efficiently.


Quality of Service (QoS):
- Quality of Service (QoS) is a set of techniques used to prioritize certain types of network traffic.
- It allows network devices to allocate bandwidth and manage traffic based on predefined policies.
- QoS is commonly used to prioritize latency-sensitive applications such as voice, video, and critical business systems.


Traffic Shaping:
- Traffic shaping is a network management technique that controls the rate at which data packets are transmitted.
- It helps smooth traffic bursts and prevent network congestion by limiting or delaying certain types of traffic.
- Traffic shaping is often used by service providers and enterprise networks to maintain consistent performance across the network.


==================================================
3. Key Network Performance Metrics (Visual Overview)
==================================================

These five metrics explain most network performance problems:

Bandwidth     → How much data can travel  
Latency       → How long data takes to travel  
Throughput    → How much data actually arrives  
Jitter        → How consistent packet timing is  
Packet Loss   → How much data disappears

4. Example Performance Problems
==================================================

Slow website:
- High latency or packet loss.


Laggy video call:
- High jitter or packet loss.


Slow downloads:
- Low throughput or limited bandwidth.
