Network Infrastructure
- Physical and virtual devices responsible for forwarding, filtering, and controlling network traffic


==================================================
1. Core Devices
==================================================
- Core networking devices are responsible for moving data between systems.
- They operate at different layers of the networking stack and form the backbone of most networks.


Hub:
- A hub is a simple Layer 1 device that broadcasts incoming signals to all connected ports.
- It does not inspect or filter traffic and simply repeats electrical signals.
- Because of its inefficiency and security limitations, hubs are largely obsolete and have been replaced by switches.


Switch:
- A switch is a networking device that forwards data frames between devices within a local network.
- Unlike a hub, a switch uses MAC addresses to determine the correct destination port for each frame.
- Switches reduce collisions and improve network performance by sending traffic only where it needs to go.


Layer 2 Switch:
- A Layer 2 switch operates at the Data Link layer of the OSI model and makes forwarding decisions based on MAC addresses.
- It maintains a MAC address table to map devices to specific ports.
- Layer 2 switches are commonly used in LAN environments to connect multiple devices within the same network.


Layer 3 Switch:
- A Layer 3 switch combines switching functionality with routing capabilities.
- It can forward traffic between different IP networks or VLANs while still performing high-speed switching.
- Layer 3 switches are often used in large enterprise networks where both switching and routing are required.


Router:
- A router is a device that forwards packets between different networks using IP addresses.
- Routers analyze routing tables to determine the best path for packets to reach their destination.
- They are commonly used to connect local networks to other networks such as the Internet.


Bridge:
- A bridge is a device that connects two network segments and forwards traffic between them based on MAC addresses.
- It functions similarly to a switch but typically has fewer ports and simpler functionality.
- Bridges were historically used to divide networks into smaller segments.


Repeater:
- A repeater is a Layer 1 device that regenerates or amplifies signals to extend the distance of a network connection.
- As signals travel through cables they weaken over distance, and repeaters help maintain signal integrity.
- Modern networking equipment often integrates repeater functionality internally.


==================================================
2. Wireless Infrastructure
==================================================
- Wireless infrastructure devices provide network connectivity using radio signals instead of physical cables.


WiFi Access Point:
- A WiFi Access Point (AP) allows wireless devices to connect to a wired network using WiFi.
- It acts as a bridge between wireless clients and the wired LAN.
- Access points are commonly used in homes, offices, and public spaces to provide wireless network coverage.


Wireless Controller:
- A wireless controller manages multiple access points from a central location.
- It handles tasks such as configuration, security policies, channel management, and client roaming.
- Wireless controllers are typically used in large enterprise wireless networks.


Wireless Bridge:
- A wireless bridge connects two separate wired networks using a wireless link.
- This is useful when running physical cables is difficult or expensive.
- Wireless bridges are commonly used to connect buildings or remote network segments.


==================================================
3. Traffic Control Devices
==================================================
- Traffic control devices inspect, filter, or manage network traffic to improve security, performance, and reliability.


Firewall:
- A firewall monitors and filters network traffic based on predefined security rules.
- It can allow or block traffic depending on IP addresses, ports, protocols, or application behavior.
- Firewalls are commonly placed at network boundaries to protect internal systems from external threats.


Load Balancer:
- A load balancer distributes incoming network traffic across multiple servers.
- This improves performance, scalability, and reliability by preventing any single server from becoming overloaded.
- Load balancers are widely used in web infrastructure and cloud environments.


Proxy:
- A proxy server acts as an intermediary between a client and another server.
- Instead of connecting directly to a destination, the client sends requests to the proxy, which forwards them.
- Proxies can provide caching, filtering, anonymity, and access control.


Reverse Proxy:
- A reverse proxy sits in front of one or more servers and forwards incoming client requests to them.
- It hides the internal server architecture and can provide load balancing, caching, and security protections.
- Reverse proxies are commonly used for web servers and APIs.


NAT Device:
- A Network Address Translation (NAT) device modifies IP address information in packets as they pass through a router or firewall.
- This allows multiple devices on a private network to share a single public IP address.
- NAT is widely used in home and enterprise networks to conserve IPv4 addresses.


VPN Gateway (Virtual Private Network):
- A VPN gateway establishes secure encrypted tunnels between networks or remote users.
- It allows secure communication over public networks such as the Internet.
- VPN gateways are used for remote access VPNs and site-to-site VPN connections.


IDS (Intrusion Detection System):
- An IDS monitors network traffic for suspicious or malicious activity.
- It analyzes traffic patterns and generates alerts when potential attacks or anomalies are detected.
- IDS systems typically do not block traffic but notify administrators of possible threats.


IPS (Intrusion Prevention System):
- An IPS is similar to an IDS but can actively block or prevent malicious traffic in real time.
- It analyzes packets and can automatically stop attacks such as exploits or suspicious behavior.
- IPS systems are often deployed inline within network traffic paths.
