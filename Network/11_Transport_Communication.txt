Transport Communication
- How reliable end-to-end communication is established between hosts


==================================================
1. TCP Concepts
==================================================
- Transmission Control Protocol (TCP) provides reliable, ordered, and error-checked delivery of data between applications running on different hosts.
- It is a connection-oriented protocol, meaning a communication session must be established before data is transmitted.


Three-Way Handshake:
- The three-way handshake is the process TCP uses to establish a connection between two hosts before data transmission begins.
- It involves three steps: the client sends a SYN packet to initiate the connection, the server responds with a SYN-ACK packet to acknowledge and synchronize, and the client replies with an ACK packet.
- This process ensures both sides are ready to communicate and agree on initial sequence numbers.


Flow Control:
- Flow control ensures that a sender does not overwhelm a receiver with more data than it can process.
- TCP uses mechanisms such as the receive window to limit how much data can be sent before receiving an acknowledgement.
- This helps maintain efficient communication and prevents buffer overflow at the receiving host.


Congestion Control:
- Congestion control regulates the rate at which data is transmitted to avoid overwhelming the network.
- TCP dynamically adjusts its transmission rate based on network conditions using algorithms such as slow start and congestion avoidance.
- This helps maintain stability and fairness across shared network resources.


Reliable Transmission:
- TCP provides reliable data delivery by ensuring that packets arrive intact and in the correct order.
- Each segment is assigned a sequence number, and the receiver acknowledges received segments.
- If packets are lost or arrive out of order, TCP retransmits missing segments and reassembles the data correctly.


==================================================
2. UDP Concepts
==================================================
- User Datagram Protocol (UDP) is a lightweight transport protocol designed for speed and efficiency.
- It does not establish connections or guarantee delivery, making it suitable for applications where low latency is more important than reliability.


Connectionless Communication:
- UDP is a connectionless protocol, meaning data is sent without establishing a session between sender and receiver.
- Packets, called datagrams, are transmitted independently without acknowledgements or retransmissions.
- This reduces overhead and allows faster communication.


Low Latency Communication:
- UDP is commonly used in applications that require minimal delay, such as real-time streaming, online gaming, voice communication, and DNS queries.
- Because it avoids connection setup and reliability checks, UDP can transmit data with very low latency.


==================================================
3. Session Concepts
==================================================
- Sessions represent ongoing communication exchanges between applications running on different hosts.
- They maintain context for interactions that may involve multiple packets or requests.


Sessions:
- A session refers to the logical communication exchange between two systems during a network interaction.
- Sessions can persist for a short period (such as a web request) or longer durations (such as a remote login session).
- Protocols and applications track sessions to maintain continuity in communication.


State Management:
- State management refers to how systems track information about ongoing sessions.
- Some protocols maintain state information to track connections, authentication status, or transaction progress.
- Stateless communication does not track previous interactions, while stateful communication keeps context about the session.

4. Transport Protocol Comparison (Visual Overview)
==================================================

TCP vs UDP

TCP
- Connection-oriented
- Reliable delivery
- Ordered packets
- Retransmits lost packets
- Flow control
- Congestion control
- Higher overhead
- Used for: HTTP, HTTPS, SSH, FTP, email

UDP
- Connectionless
- No delivery guarantee
- Packets may arrive out of order
- No retransmission
- Very low overhead
- Lower latency
- Used for: DNS, VoIP, streaming, gaming
